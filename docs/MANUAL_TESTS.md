# Ручные тест-кейсы

Дополнение к автотестам. Кейсы можно завести в Allure TestOps (#592) и связать с автоматизацией.

| ID | Слой | Сценарий | Ожидание |
|----|------|----------|----------|
| M-API-01 | API | Создать доску через REST, проверить в ответе `id` и `url` | 200, валидный JSON |
| M-WEB-01 | Web | Открыть публичную доску по ссылке без логина | Доска видна, нет редиректа на login |
| M-WEB-02 | Web | Открыть карточку с доски | Card back, название совпадает |
| M-MOB-01 | Mobile | Вход в приложение, экран досок | Вкладка Boards / Доски |
| M-MOB-02 | Mobile | Deep link на доску из API | Доска открыта в приложении |
| M-MOB-03 | Mobile | Создать карточку на доске | Карточка видна после refresh |
| M-INT-01 | E2E | API создаёт доску → UI по URL → mobile deep link | Одни и те же данные на всех слоях |

Автоматизация покрывает строки M-API-01, M-WEB-01/02, M-MOB-01/02 и M-INT-01 (частично через API-first сценарии).
